# JSON, Parquet и NPZ

## Назначение

Сериализатор сохраняет статистические параметры и числовые массивы по [контракту артефакта модели](../../contracts/model-artifact.md). JavaScript Object Notation (JSON) используется для структурированных метаданных и параметров. Apache Parquet используется для табличных параметров. NumPy Zip (NPZ) используется для числовых массивов без объектов.

## Применение

Реализация выбирает формат по форме данных модели. JSON подходит для небольших структурированных значений, Parquet — для таблиц, NPZ — для числовых массивов. Список созданных файлов передаётся издателю артефакта.

## Ограничения

NPZ не содержит объектные массивы. Допустимые файлы, их размещение и правила проверки определяет контракт артефакта модели.
