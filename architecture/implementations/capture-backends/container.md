# Контейнерный механизм захвата

## Назначение

Реализация запускает одно контролируемое приложение в контейнере. Среда выполнения контейнера предоставляет network namespace — сетевое пространство имён Linux — и control group version 2 (cgroup v2).

## Преимущества

- Сетевая и процессная изоляция создаются стандартной средой выполнения контейнера.
- Ограничения ресурсов и очистка имеют готовые механизмы.
- Образ фиксирует окружение контролируемого приложения.

## Недостатки

- Файловая система и окружение контейнера отличаются от локального запуска.
- Нужны образ и обязательная контейнерная зависимость.
- Произвольные локальные команды и Text User Interface (TUI), текстовый пользовательский интерфейс, требуют дополнительных монтирований, терминала и настройки пользователя.

## Статус

Механизм подходит для приложений с подготовленным образом, но не является универсальным вариантом первой версии. Общий жизненный цикл и публикация определены [стадией захвата](../../stages/10-capture/README.md).
