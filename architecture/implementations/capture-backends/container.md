# Контейнерный backend

## Назначение

Реализация запускает одно контролируемое приложение в контейнере. Контейнерный runtime предоставляет network namespace — сетевое пространство имён Linux — и control group version 2 (cgroup v2).

## Преимущества

- Сетевая и процессная изоляция создаются стандартным runtime.
- Ограничения ресурсов и очистка имеют готовые механизмы.
- Образ фиксирует окружение контролируемого приложения.

## Недостатки

- Файловая система и runtime-окружение отличаются от локального запуска.
- Нужны образ и обязательная контейнерная зависимость.
- Произвольные локальные команды и Text User Interface (TUI), текстовый пользовательский интерфейс, требуют дополнительных mount, terminal и user-настроек.

## Статус

Backend подходит для приложений с подготовленным образом, но не является универсальным вариантом первой версии. Общий жизненный цикл и публикация определены [стадией захвата](../../stages/10-capture/README.md).
