# eBPF-фильтрация по cgroup

## Назначение

Extended Berkeley Packet Filter (eBPF) связывает сетевые события с control group version 2 (cgroup v2), не перемещая приложение в отдельную сетевую среду.

Эта схема не создаёт обязательные для текущего CAP-контракта network namespace (`CAP-002`) и выделенный `veth` (`CAP-005`). Поэтому она не реализует CAP-контракт первой версии, недоступна для первой версии и не является взаимозаменяемой реализацией текущего backend. Её включение потребует новой версии контракта с другими требованиями к изоляции и границе захвата.

## Преимущества

- Приложение остаётся в обычном сетевом окружении.
- Наблюдаемые адреса и интерфейсы меняются меньше.
- События можно связать с cgroup дерева процессов.

## Недостатки

- Нужен код, зависящий от ядра, и отдельная сборка Berkeley Packet Filter (BPF) программ.
- Выше риск потери событий или неверной классификации пакетов.
- Совместимость зависит от версий ядра Windows Subsystem for Linux (WSL).
- Проверка двунаправленного потока и потомков существенно сложнее.

## Статус

Backend не входит в первую версию. До проектирования новой версии CAP-контракта и его включения нужны отдельные тесты полноты, классификации, потерь и переносимости.
