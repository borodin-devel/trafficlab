# Контракт извлекателя признаков

## Назначение

Feature extractor — извлекатель признаков — преобразует пакет набора данных или его ограниченные окна в таблицу признаков. Этот документ задаёт общую точку расширения. Конкретные реализации пока не выбраны и не объявляются.

## Интерфейс

Извлекатель получает:

- читатель пакета по [контракту набора данных](../../contracts/dataset.md);
- нормализованные параметры окон и агрегации;
- явное начальное значение генератора псевдослучайных чисел (`seed`), если преобразование использует выборку;
- требуемую моделью версию схемы признаков.

Результат состоит из таблицы признаков и машиночитаемой схемы признаков. Преобразование не обращается к стадии захвата, не читает глобальную конфигурацию и не изменяет входной пакет.

## Схема признаков

Схема признаков обязана содержать:

- собственные имя и семантическую версию;
- совместимые версии схемы набора данных;
- упорядоченный список признаков, их типы, допустимость пустых значений и единицы измерения;
- параметры окон, группировки и агрегации;
- правила формирования идентификатора строки и её связи с пакетом, потоком или сессией;
- признак использования случайной выборки и способ воспроизведения через начальное значение генератора псевдослучайных чисел (`seed`);
- хеш канонического представления схемы.

Модель объявляет требуемую схему признаков до обучения. Сохранённый [артефакт модели](../../contracts/model-artifact.md) включает `feature_schema.json` в формате JavaScript Object Notation (JSON) и соответствующий хеш. Несовместимая версия прекращает выполнение до вычисления признаков.

## Регистрация

Каждая реализация регистрируется явно под стабильным именем как соответствие имени фабрике. Автоматическая регистрация через побочный эффект импорта не допускается. Внешние точки входа могут быть добавлены только после стабилизации контракта.

Реестр отклоняет неизвестное имя и повторную регистрацию. Конфигурация сохраняет имя реализации, её версию и нормализованные параметры.

## Инварианты

- Один вход, одна схема, одни параметры и одно начальное значение генератора псевдослучайных чисел (`seed`) дают одинаковую таблицу признаков.
- Порядок выходных столбцов совпадает с порядком схемы.
- Извлечение выполняется пакетами и не требует загрузки всего набора в память.
- Признаки не меняют принадлежность целой сессии обучающей, проверочной или тестовой части.
- Реализация не добавляет скрытые признаки, отсутствующие в сохранённой схеме.
