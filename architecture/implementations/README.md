# Каталог реализаций

Этот раздел содержит взаимозаменяемые реализации контрактов. Стадии определяют процесс и требования, контракты — формат обмена, а файлы здесь — устройство, ограничения и статус конкретного инструмента или алгоритма.

## Семейства

- [Механизмы захвата](capture-backends/) — реализации механизма захвата (`capture backend`) по интерфейсу `CaptureBackend`.
- [Декодеры пакетов](packet-decoders/) — реализации `PacketDecoder`.
- [Средства построения пакетов](packet-renderers/) — реализации `PacketRenderer`.
- [Извлекатели признаков](feature-extractors/) — реализации `FeatureExtractor`.
- [Модели](models/) — реализации `ModelPlugin`.
- [Генераторы](generators/) — реализации `GeneratorPlugin` и адаптер активной эмуляции.
- [Метрики](metrics/) — реализации `MetricPlugin`.
- [Валидаторы](validators/) — реализации валидаторов захвата, `DatasetValidator` и валидатора синтетического трафика.
- [Оптимизаторы](optimizers/) — реализации `OptimizerPlugin`.
- [Сериализаторы артефактов](artifact-serializers/) — реализации `ArtifactSerializer`.

Все реализации одного семейства находятся на одном уровне соответствующего каталога. Конкретный файл хранит собственные метаданные статуса, ссылку на контракт и список использующих стадий; каталог не повторяет его описание.

## Регистрация

Первая версия использует только явный реестр. Имя из конфигурации сопоставляется с фабрикой в `registry.py`, например:

```text
"netns_dumpcap" -> create_netns_dumpcap_backend
"tshark"        -> create_tshark_decoder
"scapy"         -> create_scapy_renderer
"semi_markov"   -> create_semi_markov_model
```

Автоматическая регистрация как побочный эффект импорта запрещена. Добавление реализации требует явной записи в реестре и проверки совместимости её контракта. Внешние точки входа Python (entry points) могут появиться только после стабилизации контрактов и не входят в первую версию.
