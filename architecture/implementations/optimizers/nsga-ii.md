# Non-dominated Sorting Genetic Algorithm II

## Назначение

Non-dominated Sorting Genetic Algorithm II (NSGA-II) — генетический алгоритм с недоминируемой сортировкой для многокритериальной оптимизации. Реализация выбирает кандидатов по `quality_vector`, сохраняя варианты с разными компромиссами между семействами метрик.

## Входы и результат

Оптимизатор получает текущую популяцию, потомков, признаки допустимости, `quality_vector`, пространство поиска и настройки генетических операторов. Порядок и направление компонентов вектора задают [контракт результата метрики](../../contracts/metric-result.md) и [правила многокритериальной оценки](../../stages/70-compare/evaluation.md).

Результат содержит популяцию следующего поколения, ранг фронта Парето, crowding distance — расстояние скученности — и выбранные пары родителей. `weighted_score` может использоваться внешней стадией для отображения или условия завершения, но не участвует в недоминируемой сортировке NSGA-II.

## Недоминируемая сортировка

Кандидат A доминирует кандидата B, если A не хуже B по каждой компоненте `quality_vector` и строго лучше хотя бы по одной. Перед сравнением каждая компонента уже приведена профилем метрик к объявленному направлению качества.

Алгоритм для каждого кандидата строит множество доминируемых им кандидатов и считает число кандидатов, которые доминируют его. Кандидаты с нулевым счётчиком образуют первый фронт. После их удаления счётчики доминируемых кандидатов уменьшаются; новые нулевые значения образуют следующий фронт. Процесс продолжается до распределения всей допустимой популяции.

`hard_valid` является ограничением допуска, а не шестой целью. Допустимый кандидат всегда предпочтительнее недопустимого. Два недопустимых кандидата можно упорядочить только по явно заданной величине нарушения ограничения; отсутствующий или ошибочный `quality_vector` не заменяется фиктивными наилучшими значениями.

## Отбор по Парето

Родительская и дочерняя популяции объединяются. Фронты добавляются в новую популяцию по возрастанию ранга, пока очередной фронт помещается целиком. Если мест меньше, чем кандидатов в последнем фронте, выбор выполняется по убыванию расстояния скученности.

Расстояние скученности оценивает изолированность кандидата внутри одного фронта. Для каждой цели кандидаты сортируются по значению, граничные точки получают приоритет, а внутренние получают нормированную сумму расстояний до соседей. Нулевая ширина цели не добавляет расстояние. Это правило сохраняет разнообразие вдоль фронта Парето без сведения целей к одной оценке.

Для выбора родителей применяется бинарный турнир. Сначала сравнивается ранг фронта, затем расстояние скученности. Полная ничья разрешается воспроизводимым псевдослучайным выбором с сохранённым seed — исходным значением генератора псевдослучайных чисел.

## Скрещивание и мутация

Crossover — скрещивание — создаёт потомка из двух генотипов. Оператор выбирается по типу параметра: категориальные поля наследуются от одного родителя, числовые могут смешиваться в разрешённом диапазоне, а зависимые группы параметров переносятся атомарно. После операции генотип приводится к канонической форме и проверяется на ограничения пространства поиска.

Mutation — мутация — изменяет отдельные поля генотипа с заданной вероятностью. Категориальное поле получает другое допустимое значение, числовое — ограниченное возмущение или новую выборку из диапазона, а структурное изменение одновременно обновляет зависимые поля. Необработанные веса нейронной сети не являются генами и этими операторами не изменяются.

Каждая операция сохраняет genotype hash — хеш генотипа — родителей, параметры оператора и seed. Недопустимый по статическим ограничениям потомок отклоняется до дорогостоящей оценки либо исправляется только явно настроенным детерминированным оператором восстановления. Создание дубликата обнаруживается по каноническому genotype hash и не требует повторной оценки.

## Ограничения

NSGA-II не определяет общий цикл обучения, генерации, проверки и сравнения; им владеет [стадия эволюционного поиска](../../stages/80-evolve/README.md). Алгоритм не гарантирует единственного лучшего кандидата: результатом является аппроксимация фронта Парето. Качество поиска зависит от размера популяции, бюджета поколений, операторов и границ пространства поиска.
