# Валидатор набора данных

## Назначение

Реализация проверяет пакет Apache Parquet и его метаданные перед обучением. Состав и смысл проверок принадлежат каноническим [правилам исходного набора](../../stages/30-validate-source/rules.md).

## Выполнение

Валидатор сначала разбирает манифест и схему, затем последовательно читает таблицы ограниченными группами строк. Проверки запускаются по стабильным идентификаторам из `rules.md`. Для межтабличных проверок реализация хранит только необходимые идентификаторы, счётчики, границы и агрегаты.

Профиль задаёт разрешённые версии, пороги и серьёзность настраиваемых правил. Фактическое значение профиля записывается в отчёт. Реализация не переопределяет обязательные правила и не считает неизвестную версию совместимой.

## Результат

Каждое выполненное правило создаёт запись `rules[]` по [контракту отчёта валидации](../../contracts/validation-report.md). Свидетельство содержит наблюдаемое значение, ожидание и короткую причину отказа. Итоговые счётчики и статус вычисляются из результатов правил, а не задаются отдельной ветвью логики.

Ошибка чтения проверяемого файла оформляется как непройденное правило, если файл был корректно открыт как вход проверки. Сбой самой программы или невозможность сформировать полный отчёт является ошибкой стадии и не выдаётся за результат `fail`.

## Ограничения

- Валидатор не исправляет данные и не изменяет проверяемый пакет.
- Валидатор не определяет новые поля набора данных.
- Поведенческая реалистичность трафика не входит в проверку исходного набора.
- Полный набор строк не помещается в отчёт; сохраняются только ограниченные свидетельства и агрегаты.
