# Первичная проверка пакетного захвата

## Назначение

Валидатор проверяет файлы Packet Capture Next Generation (PCAPNG) после штатного закрытия. Он различает структурную корректность файла и пригодность данных для последующих стадий. Условия публикации принадлежат [стадии захвата](../../stages/10-capture/publication.md).

## Структурная корректность

`capinfos` должен успешно открыть файл и подтвердить формат PCAPNG. Валидатор получает размер, число пакетов, длительность, канальную инкапсуляцию, описания интерфейсов, `snaplen`, временной порядок и Secure Hash Algorithm 256-bit (SHA-256).

Полученные интерфейсы и `snaplen` сверяются с манифестом. Файл должен существовать, превышать минимальный структурный размер и полностью читаться. Ошибка чтения, неизвестный обязательный интерфейс или несогласованная конфигурация дают `structurally_valid = false`.

## Дополнительная проверка

`tshark` последовательно читает все записи и проверяет разбор доступных полей Layer 2 (L2), Layer 3 (L3) и Layer 4 (L4), канального, сетевого и транспортного уровней. Для каждой записи захваченная длина не превышает исходную, а временная метка разбирается.

Ошибочный checksum не считается повреждением автоматически. Валидатор учитывает фактические transmit (TX) и receive (RX) checksum offload из манифеста.

## Пригодность набора данных

Структурно корректный файл может быть непригоден для анализа. Например, нулевое или меньшее заданного минимума число пакетов не означает повреждение контейнера PCAPNG, но не даёт достаточного набора данных.

Результат хранит независимые признаки и машиночитаемую причину:

```json
{
  "structurally_valid": true,
  "dataset_usable": false,
  "reason": "packet_count_below_minimum"
}
```

Порог числа пакетов и обязательные семейства протоколов задаются конфигурацией стадии. Валидатор не решает, публиковать ли диагностический остаток.

## Инварианты

- Проверяется только закрытый файл.
- Структурная корректность не выводится только из размера или числа пакетов.
- Пригодность данных не маскирует структурную ошибку.
- Хеш вычисляется для тех же байтов, которые прошли проверку.
