# Валидатор синтетического трафика

## Назначение

Реализация проверяет составной синтетический артефакт с packet events — абстрактными пакетными событиями — и применимыми материализованными компонентами. Состав, идентификаторы и смысл проверок принадлежат каноническим [правилам синтетического трафика](../../stages/60-validate-synthetic/rules.md).

## Выполнение

Валидатор разбирает `synthetic-manifest.json`, метаданные событий, запрос генерации и один из трёх составных профилей: `synthetic-events-v1`, `synthetic-rendered-v1` или `synthetic-active-v1`. Компоненты он открывает только по манифесту. До запуска правил он сверяет набор компонентов с выбранным профилем: отсутствующий обязательный компонент даёт `fail`, а не `skip`.

Для `synthetic-rendered-v1` он последовательно читает `rendered_pcapng`; для `synthetic-active-v1` — `active_capture` и свидетельство изоляции. Пакеты сопоставляются с событиями. Для незавершённых потоков валидатор хранит ограниченное протокольное состояние; для сессий и временных окон — счётчики и агрегаты. В `synthetic-events-v1` отсутствующие байты пакетов не синтезируются: соответствующие байтовые правила получают `skip` с причиной `component_not_present`.

Проверки запускаются по стабильным идентификаторам и матрице применимости из `rules.md`. Профиль передаёт только разрешённые параметры и не переопределяет обязательность правила. Неприменимость оформляется как `skip` с машиночитаемой причиной; сетевое правило в `synthetic-active-v1` неприменимым быть не может.

Вторая ветвь валидации запускает [валидатор набора данных](dataset.md) с профилем `synthetic-dataset-v1` на `components.dataset`, а затем проверяет его происхождение по `SYNTH-DATASET-SOURCE-001`. Реализация повторно использует владеющий контракт и канонические dataset-правила; она не определяет копию схемы.

## Результат

Каждая выполненная проверка создаёт запись `rules[]` по [контракту отчёта валидации](../../contracts/validation-report.md). Составная ветвь записывает `validation.json` с `validated_artifact_id`, равным идентификатору составного артефакта. Dataset-ветвь записывает `dataset-validation.json` с профилем `synthetic-dataset-v1` и `validated_artifact_id`, равным `components.dataset.artifact_id`. Итоговый статус и счётчики каждого отчёта вычисляются независимо из его результатов правил. Свидетельства ограничены агрегатами, индексами и короткими причинами; полезная нагрузка и полные последовательности в отчёты не копируются.

## Ограничения

- Валидатор не исправляет события и пакеты.
- Валидатор не определяет параллельный набор правил или порогов.
- Проверяемый PCAPNG не загружается в память целиком.
- Сбой программы или невозможность сформировать полный отчёт является ошибкой стадии, а не результатом `fail`.
