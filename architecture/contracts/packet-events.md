# Контракт пакетных событий

## Назначение

Packet event — абстрактное пакетное событие. Модель и случайный генератор создают последовательность таких событий, не формируя байты пакета. Контракт не определяет устройство модели, планировщика или средства построения пакетов.

Версия последовательности задаётся полем `schema_version`. Начальная версия — `1.0.0`; она изменяется по правилам семантического версионирования. Поле `l4_protocol` обозначает протокол транспортного уровня — Layer 4 (L4).

## Обязательные поля

```text
trace_id: string
session_id: string
flow_id: string
packet_index: uint64
relative_time_ns: int64
inter_arrival_ns: int64
direction: enum[ingress, egress, unknown]
original_length: uint32
l4_protocol: enum[tcp, udp, icmp, icmpv6, other, none]
src_endpoint_id: string
dst_endpoint_id: string
```

Протокольные расширения являются nullable-полями:

```text
ether_type: uint16?
ip_version: uint8?
ip_protocol: uint8?
tcp_flags: uint16?
src_port_class: uint16?
dst_port_class: uint16?
```

Метаданные последовательности обязаны содержать `artifact_type = packet_events`, `artifact_version`, `schema_version`, `artifact_id`, `created_at`, `seed`, `generator`, `event_count` и `lineage`.

## Составной синтетический артефакт

Результат генерации публикуется с манифестом в формате JavaScript Object Notation (JSON) — `synthetic-manifest.json`. Один `artifact_id` манифеста задаёт идентичность всего проверяемого результата и перечисляет компоненты с их идентификаторами или SHA-256-хешами (Secure Hash Algorithm 256, 256-битный алгоритм защищённого хеширования):

```json
{
  "schema_version": "1.0.0",
  "artifact_type": "synthetic_traffic",
  "artifact_id": "sha256:...",
  "components": {
    "packet_events": {"path": "events.jsonl", "artifact_id": "sha256:...", "sha256": "sha256:..."},
    "dataset": {"path": "dataset/", "artifact_id": "sha256:...", "sha256": "sha256:..."},
    "rendered_pcapng": {"path": "synthetic.pcapng", "sha256": "sha256:..."},
    "active_capture": null,
    "isolation_evidence": null
  }
}
```

`packet_events` и `dataset` обязательны для результата, передаваемого сравнению. Компонент `dataset` ссылается по `artifact_id` и SHA-256-хешу на пакет по [контракту набора данных](dataset.md), детерминированно построенный из `packet_events`. `rendered_pcapng` присутствует после автономной материализации, а `active_capture` и `isolation_evidence` присутствуют вместе после активной эмуляции. Отсутствующий неприменимый компонент имеет значение `null`; каждый присутствующий компонент содержит относительный `path` и идентификатор или хеш. `artifact_id` равен SHA-256 канонического JSON-представления манифеста без самого поля `artifact_id` и всех полей `path`: ключи сортируются лексикографически, JSON кодируется в UTF-8 (Unicode Transformation Format, 8-bit — 8-битный формат преобразования Unicode) без незначащих пробелов. Поэтому размещение файлов не меняет идентичность их содержимого.

Отчёт синтетической проверки `validation.json` указывает этот составной `artifact_id` в `validated_artifact_id`. Валидатор составного артефакта принимает манифест и разрешает компоненты только через него, а не сопоставляет независимые входные файлы. Отдельный `dataset-validation.json` проверяет компонент `dataset` и указывает `components.dataset.artifact_id`.

## Свидетельство изоляции активного режима

Компонент `isolation_evidence` ссылается на `isolation-evidence.json` и его SHA-256-хеш. Файл обязателен для `active-emulation` и содержит:

- топологию, идентификаторы endpoint namespace, интерфейсов, bridge или виртуальных Ethernet-связей;
- установленные маршруты и явное подтверждение отсутствия default и внешних маршрутов;
- состояние IPv4/IPv6 forwarding и подтверждение отсутствия NAT;
- состояние boundary firewall с политикой `default-deny`;
- счётчики принятых и отклонённых пакетов на внешней границе, включая отдельный счётчик принятого external egress;
- результат проверки изоляции и временную метку;
- SHA-256-хеши относящихся к запуску итоговых конфигураций.

Содержимое свидетельства должно подтверждать [fail-closed границу активной эмуляции](../common/security.md#сетевая-безопасность-генерации). Отсутствующее или противоречивое свидетельство и ненулевой счётчик принятого external egress делают активный составной артефакт недействительным.

## Инварианты

- `packet_index` начинается с нуля и строго возрастает внутри trace.
- `relative_time_ns` не убывает; `inter_arrival_ns` неотрицателен.
- `event_count` совпадает с числом событий.
- Идентификаторы endpoint являются псевдонимами и не указывают случайно на реальные внешние узлы.
- События проходят независимую семантическую валидацию до материализации.
- Длина, контрольные суммы и зависимые поля пакета не являются обязанностью события.
- Одинаковый вход, конфигурация и `seed` дают одинаковую последовательность.

## Потребители

Контракт последовательности создают модели и генераторы событий; составной манифест создаёт стадия генерации. Последовательность читают семантический валидатор, средство построения пакетов, автономный синтез и активная эмуляция. Эволюционный оркестратор разрешает компонент `dataset` и передаёт его вместе с успешным `dataset-validation.json` стадии сравнения; сама стадия сравнения составной манифест не читает.
