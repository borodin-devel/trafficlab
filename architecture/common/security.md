# Безопасность

## Граница привилегий

Основная программа работает от обычного пользователя. Отдельный helper получает повышенные права только для создания и удаления сетевого пространства имён Linux, виртуальных Ethernet-интерфейсов, адресов, маршрутов, Network Address Translation (NAT), Domain Name System (DNS), control group version 2 (cgroup v2), а также для запуска захвата и помещения приложения в подготовленное окружение.

Перед запуском целевого приложения helper сбрасывает права root и возвращается к исходным User Identifier (UID) и Group Identifier (GID). Целевое приложение никогда не работает от root. Вся основная программа не запускается через `sudo`.

## Граница изоляции

Network namespace изолирует сетевые интерфейсы, маршруты, стек, межсетевой экран (firewall) и сокеты. Он не изолирует файловую систему, процессы, Inter-Process Communication (IPC), пользователей и системные вызовы. Это не sandbox для недоверенного приложения.

Первая версия не гарантирует защиту от привилегированного или вредоносного приложения, способного выполнить `setns`, изменить cgroup или сетевую конфигурацию. Для такого сценария нужны дополнительные namespaces, seccomp и отдельная файловая изоляция.

## Данные и десериализация

Полезная нагрузка пакетов по умолчанию не сохраняется в наборе данных. Ограниченная длина захвата уменьшает объём чувствительных данных, но не заменяет контроль доступа.

Секреты, payload и credentials переменных окружения маскируются в журналах. Межпрограммные артефакты модели не используют Python pickle или joblib. Локальный недоверенный pickle разрешено загружать только в совпадающем зафиксированном окружении.

## Сетевая безопасность генерации

Синтетические адреса не должны случайно указывать на реальные внешние узлы. По умолчанию разрешены только:

- сети документации интернет-протокола версии 4 — Internet Protocol version 4 (IPv4) — `192.0.2.0/24`, `198.51.100.0/24` и `203.0.113.0/24` из документа серии «Запрос комментариев» — Request for Comments (RFC) 5737;
- сеть документации интернет-протокола версии 6 — Internet Protocol version 6 (IPv6) — `2001:db8::/32` из RFC 3849;
- локально администрируемые одноадресные адреса управления доступом к среде — Media Access Control (MAC).

Профиль может выбрать только подмножество этого списка. Произвольные публичные, частные, loopback, link-local, multicast и адреса хоста запрещены, пока отдельная безопасная политика будущей версии явно не разрешит их.

Активная эмуляция работает в режиме закрытого отказа (fail-closed) в виртуальной сети с двумя контролируемыми конечными сетевыми узлами (endpoint). Их сетевые пространства имён соединяются только изолированным сетевым мостом (bridge) или прямыми виртуальными Ethernet-парами. Топология не имеет восходящего соединения (uplink) с интерфейсами хоста или внешней сети; маршруты по умолчанию и внешние маршруты отсутствуют. Network Address Translation (NAT) и пересылка пакетов (forwarding) на интерфейсы хоста или внешней сети запрещены, а политика межсетевого экрана на границе — запрет по умолчанию (`default-deny`). Пакеты не отправляются через реальный интерфейс Windows или другого хоста.

До запуска endpoint контроллер проверяет интерфейсы, маршруты, состояние forwarding и firewall, отсутствие NAT и uplink. Во время запуска отдельный контроллер границы (boundary guard) сохраняет счётчики принятых и отклонённых пакетов на внешней границе. Запуск недействителен, если проверка или её свидетельство отсутствует, противоречиво либо сообщает хотя бы один принятый пакет исходящего трафика во внешнюю сеть (external egress). Формат обязательного свидетельства и его включение в составной артефакт задаёт [контракт пакетных событий и синтетического артефакта](../contracts/packet-events.md#свидетельство-изоляции-активного-режима).

## Инварианты

- Повышенные права ограничены минимальным helper и временем операции.
- Пути, команды и аргументы проходят проверку до привилегированного вызова.
- Команда приложения передаётся массивом аргументов без интерпретации shell.
- Диагностические данные не раскрывают секреты и payload.
- Сетевой namespace не обозначается как полная изоляция процесса.

## Потребители

Правила обязательны для захвата, helper, оркестратора, журналирования, сериализаторов модели, генерации и активной эмуляции.
