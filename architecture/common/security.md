# Безопасность

## Граница привилегий

Основная программа работает от обычного пользователя. Отдельный helper получает повышенные права только для создания и удаления сетевого пространства имён Linux, виртуальных Ethernet-интерфейсов, адресов, маршрутов, Network Address Translation (NAT), Domain Name System (DNS), control group version 2 (cgroup v2), а также для запуска захвата и помещения приложения в подготовленное окружение.

Перед запуском целевого приложения helper сбрасывает права root и возвращается к исходным User Identifier (UID) и Group Identifier (GID). Целевое приложение никогда не работает от root. Вся основная программа не запускается через `sudo`.

## Граница изоляции

Network namespace изолирует сетевые интерфейсы, маршруты, стек, firewall и сокеты. Он не изолирует файловую систему, процессы, Inter-Process Communication (IPC), пользователей и системные вызовы. Это не sandbox для недоверенного приложения.

Первая версия не гарантирует защиту от привилегированного или вредоносного приложения, способного выполнить `setns`, изменить cgroup или сетевую конфигурацию. Для такого сценария нужны дополнительные namespaces, seccomp и отдельная файловая изоляция.

## Данные и десериализация

Полезная нагрузка пакетов по умолчанию не сохраняется в наборе данных. Ограниченная длина захвата уменьшает объём чувствительных данных, но не заменяет контроль доступа.

Секреты, payload и credentials переменных окружения маскируются в журналах. Межпрограммные артефакты модели не используют Python pickle или joblib. Локальный недоверенный pickle разрешено загружать только в совпадающем зафиксированном окружении.

## Сетевая безопасность генерации

Синтетические адреса не должны случайно указывать на реальные внешние узлы. Активная эмуляция разрешена только в изолированной виртуальной сети с двумя контролируемыми endpoint. Пакеты не отправляются через реальный интерфейс Windows.

## Инварианты

- Повышенные права ограничены минимальным helper и временем операции.
- Пути, команды и аргументы проходят проверку до привилегированного вызова.
- Команда приложения передаётся массивом аргументов без интерпретации shell.
- Диагностические данные не раскрывают секреты и payload.
- Сетевой namespace не обозначается как полная изоляция процесса.

## Потребители

Правила обязательны для захвата, helper, оркестратора, журналирования, сериализаторов модели, генерации и активной эмуляции.
