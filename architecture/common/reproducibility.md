# Воспроизводимость

## Назначение

Воспроизводимость означает возможность объяснить и повторить результат стадии по сохранённым входам, настройкам и окружению.

## Обязательные данные

Каждая стадия сохраняет:

- полную разрешённую конфигурацию;
- начальное значение генератора псевдослучайных чисел (`seed`);
- версии пакетов Python;
- версии внешних программ;
- зафиксированное изменение Git (`commit`) и состояние рабочей директории;
- хеши входных и выходных артефактов;
- системную информацию;
- режим сети Windows Subsystem for Linux (WSL);
- фактические настройки сетевой разгрузки;
- временную зону;
- региональные настройки (`locale`).

Эти данные размещаются в метаданных каталога запуска и стадии по их контрактам. Секретные значения не сохраняются; вместо них фиксируются безопасные признаки наличия или идентификаторы источников.

## Детерминизм

Конфигурация нормализуется до вычисления идентификатора стадии. Порядок входов фиксируется. Все компоненты, использующие случайность, получают явное начальное значение генератора. Детерминированная выборка для ограниченного контекста и разбиение на обучающую (`train`), проверочную (`validation`) и тестовую (`test`) части воспроизводятся по сохранённым идентификаторам сессий.

Если библиотека или аппаратная операция недетерминирована, стадия фиксирует это ограничение в метаданных. Такой результат может быть повторяемым только статистически и не должен объявляться побитово идентичным.

Для детерминированного валидатора повторная проверка одного неизменяемого артефакта одним профилем даёт тот же набор результатов.

## Проверка повторного запуска

Повторный запуск сравнивает разрешённую конфигурацию, версии реализаций, упорядоченные хеши входов и сведения об окружении. Совпадение ключа позволяет использовать проверенный кэш по правилам [жизненного цикла](artifact-lifecycle.md). Несовпадение значимого поля создаёт новый результат.

## Область применения

Правила обязательны для всех стадий, моделей, генераторов, метрик, валидаторов и эволюционного цикла.
