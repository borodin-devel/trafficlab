# Ресурсы и крупные файлы

Способ хранения определяется размером, изменчивостью и назначением файла. Ни одна система хранения не применяется ко всем бинарным данным автоматически.

## Git

Обычный Git хранит исходный код, конфигурации, машиночитаемые схемы и маленькие детерминированные фикстуры. Файл должен оставаться пригодным для обычного клонирования и проверки истории.

## Git LFS

Расширение Git Large File Storage (Git LFS) хранит небольшое число стабильных бинарных фикстур и эталонных артефактов моделей ограниченного размера. Все дампы не переносятся в Git LFS автоматически: часто изменяемая коллекция быстро увеличивает стоимость истории и загрузки.

## Внешнее хранилище

Data Version Control (DVC) или внешнее объектное хранилище используется для крупных файлов Packet Capture Next Generation (PCAPNG), полных наборов данных и больших контрольных точек моделей. Изменяемые наборы используют адресуемый по содержимому кэш.

Репозиторий хранит ссылку и манифест, но не скрывает сетевую загрузку в обычном импорте или модульном тесте. Команда получения ресурса должна быть явной и проверять результат до публикации.

## Каталог запусков

`run/` содержит локальные результаты запусков и не коммитится. Структуру результата определяет [контракт каталога запуска](../contracts/run-directory.md). Опубликованный внешний ресурс переносится из `run/` только явной операцией.

## Манифест

Каждый внешний ресурс сопровождается манифестом со следующими полями:

```text
идентификатор и версия
путь или адрес объекта
размер в байтах
SHA-256
лицензия
происхождение
дата получения
команда или процесс создания
```

SHA-256 — Secure Hash Algorithm с длиной результата 256 бит. Хеш проверяет целостность и участвует в адресации по содержимому, но не заменяет подпись доверенного издателя.

## Ограничения

- Полезная нагрузка пакетов по умолчанию не сохраняется; это решение зафиксировано в [ADR-015](decisions.md#adr-015-не-сохранять-полезную-нагрузку-по-умолчанию).
- Формат Python `pickle` не является публичным форматом артефакта; см. [ADR-016](decisions.md#adr-016-не-использовать-pickle-для-публичных-артефактов).
- Секреты, учётные данные и персональные данные не помещаются ни в одно из перечисленных хранилищ.
- Лицензия и происхождение проверяются до добавления внешнего ресурса в воспроизводимый эксперимент.
- Маленькие эталонные файлы следуют правилам [тестовых фикстур](testing.md#фикстуры).
