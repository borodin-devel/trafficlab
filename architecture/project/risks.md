# Основные риски

Риски проверяются на предварительной стадии, во время выполнения и при публикации результата. Мера считается частью архитектуры, если без неё результат нельзя интерпретировать или безопасно воспроизвести.

| Риск | Мера |
| --- | --- |
| Сетевой режим Windows Subsystem for Linux (WSL) меняет наблюдаемое поведение. | Записывать режим сети и топологию; иметь контрольные интеграционные тесты для поддерживаемых режимов. |
| Аппаратные сетевые оптимизации — offload — искажают наблюдаемые длины и сегментацию пакетов. | Проверять и отключать offload на виртуальной Ethernet-паре; сохранять запрошенное и фактическое состояние. |
| Туннелирование Domain Name System (DNS) исключает DNS-пакеты из наблюдаемого пути. | Использовать явный DNS-прокси через виртуальную Ethernet-пару и проверять его тестом. |
| Потомок переживает главный процесс. | Отслеживать control group version 2 (cgroup v2), а не один process identifier (PID); завершать всё дерево. |
| Привилегированная вспомогательная программа оставляет сетевые правила и пространства имён. | Делать очистку идемпотентной; хранить журнал созданных ресурсов; предоставить отдельную очистку осиротевших ресурсов. |
| Большие файлы захвата заполняют диск. | Задать бюджет диска, кольцевой буфер и предел размера; прекращать публикацию неполного результата. |
| Метрика переобучается на конкретную трассу. | Разделять данные по сеансам и запускам; использовать несколько независимых семейств метрик. |
| Эволюционный алгоритм занимает все ресурсы. | Использовать токены ресурсов, кэш, раннее отклонение и поэтапную оценку кандидатов. |
| Синтетический Packet Capture Next Generation (PCAPNG) корректен структурно, но нереалистичен. | Разделять структурные, семантические и поведенческие валидаторы; не считать успешное чтение достаточным. |
| Нейронная модель не помещается в память. | Одновременно запускать не более одного тяжёлого обучения, ограничить архитектуру и сначала сравнивать с базовыми моделями. |
| Небольшая Large Language Model (LLM) создаёт чрезмерно связанный код. | Использовать короткие модули, явные контракты, зеркальные тесты и записи Architecture Decision Record (ADR). |
| Ограничение канального уровня — Layer 2 (L2) — принимают за захват физической сети. | Явно указывать, что наблюдается виртуальная граница; не обещать физические L2-эффекты и сверяться с [границами системы](../system/scope.md). |
| Активная эмуляция отправляет неожиданный трафик. | Запускать её только в разрешённой изолированной топологии, применять allowlist назначения и отличать от [автономного синтеза](../stages/50-generate/README.md). |
| Внешний ресурс меняется или имеет неизвестную лицензию. | Проверять хеш, размер, происхождение и лицензию по [манифесту ресурса](assets.md#манифест). |

Владелец компонента добавляет тест меры вместе с реализацией. Неустранимое ограничение записывается в манифест запуска или отчёт проверки, а не остаётся только в журнале.
