# Конвейер системы

## Линейная карта

Граничные стадии принимают внешние входы: `00-preflight` — фактическую среду и конфигурацию, `10-capture` — команду приложения, конфигурацию и успешный результат предварительной проверки среды. Зависимые стадии принимают опубликованные и проверенные артефакты предшественников. Каждая стадия проверяет все применимые к ней входы, выполняется в отдельном процессе и публикует результат по [контракту состояния стадии](../contracts/stage.md). Числовой шаг оставляет место для будущих стадий без переименования существующих.

| Стадия | Основной вход | Основной выход |
| --- | --- | --- |
| [`00-preflight`](../stages/00-preflight/README.md) | Конфигурация, выбранная реализация и фактическая среда | Отчёт о готовности среды |
| [`10-capture`](../stages/10-capture/README.md) | Команда приложения и успешная предварительная проверка | Закрытый и проверенный захват Packet Capture Next Generation (PCAPNG) |
| [`20-convert`](../stages/20-convert/README.md) | Захват PCAPNG или зарегистрированный внешний файл | Опубликованный пакет исходного набора данных |
| [`30-validate-source`](../stages/30-validate-source/README.md) | Неизменяемый исходный набор и профиль правил | Отдельный отчёт допуска к обучению |
| [`40-train`](../stages/40-train/README.md) | Допущенный набор, тип модели, исходное значение генератора псевдослучайных чисел и бюджет ресурсов | Один или несколько артефактов модели |
| [`50-generate`](../stages/50-generate/README.md) | Модель либо проверенный набор и запрос генерации | Составной синтетический артефакт с пакетными событиями и производными компонентами |
| [`60-validate-synthetic`](../stages/60-validate-synthetic/README.md) | Синтетический манифест, запрос и профиль проверки | Отчёт составной проверки и отдельный отчёт синтетического набора |
| [`70-compare`](../stages/70-compare/README.md) | Проверенные исходный и синтетические наборы | Результаты метрик, сравнение и отчёт |
| [`80-evolve`](../stages/80-evolve/README.md) | Исходный набор, пространство поиска и критерии | История поколений, фронт Парето и выбранные кандидаты |

Обычный полный запуск следует порядку `00-preflight` → `10-capture` → `20-convert` → `30-validate-source` → `40-train` → `50-generate` → `60-validate-synthetic` → `70-compare`. `80-evolve` управляет повторными оценками кандидатов после подготовки исходного набора.

## Варианты пути данных

Стадию `20-convert` можно начать с зарегистрированного внешнего PCAPNG. Внешний вход получает собственную запись происхождения и не обходит проверку исходного набора.

В автономном синтезе `50-generate` преобразует проверенные пакетные события прямо в пакет набора данных. Этот быстрый путь не требует промежуточного PCAPNG для `60-validate-synthetic` и `70-compare`. Материализация PCAPNG включается только для профиля, который проверяет реальные байты пакетов.

Стадия `60-validate-synthetic` выбирает один из трёх составных профилей: `synthetic-events-v1`, `synthetic-rendered-v1` или `synthetic-active-v1`. Каждый профиль требует свой набор компонентов; отсутствие обязательного компонента является ошибкой. Точная матрица принадлежит [документу стадии](../stages/60-validate-synthetic/README.md#профили).

Активная эмуляция является адаптером уже проверенных событий внутри `50-generate`. После активного выполнения новый составной артефакт с захватом и свидетельством изоляции снова проходит `60-validate-synthetic` с профилем `synthetic-active-v1`. Непроверенные события не попадают в активную сеть.

## Цикл эволюции

`80-evolve` сохраняет генотип и границу поколения, затем планирует для каждого кандидата нужные узлы `40-train`, `50-generate`, `60-validate-synthetic` и `70-compare`. Результаты сравнения определяют недоминируемое ранжирование, отбор, мутацию и скрещивание следующего поколения.

Обратного ребра в ориентированном ациклическом графе — Directed Acyclic Graph (DAG) — нет. Каждое поколение представлено новыми явно сохранёнными задачами. Проверенный кэш может исключить повторное вычисление совпадающего кандидата, но не отменяет проверку контрактов и хешей.

Команды полного, частичного и возобновлённого выполнения приведены в [сценариях запуска](scenarios.md).
