# Обзор системы

## Назначение

Trafficlab — исследовательский комплекс для захвата трафика Linux-приложения, построения нормализованного набора данных, обучения моделей и оценки синтетического трафика. Итоговая архитектура — модульный монолит в одном репозитории и одном Python-дистрибутиве.

## Архитектурное решение

Пользовательские операции разделены между независимыми [консольными программами](programs.md). Они используют общие [внутренние модули](modules.md), но обмениваются только версионированными файловыми артефактами. Форматы обмена определены в [контрактах](../contracts/), поэтому стадия не читает внутреннее состояние другой стадии.

Главная программа строит ориентированный ациклический граф — Directed Acyclic Graph (DAG). Узлы графа выполняются отдельными процессами, публикуют неизменяемые результаты и могут повторно использовать проверенный кэш. Общие правила запуска, возобновления и частичного выполнения определяет [оркестрация](../common/orchestration.md).

Код следует стилю «функциональное ядро, императивная оболочка» — functional core, imperative shell. Чистые преобразования и решения отделены от файлового ввода-вывода, запуска процессов, сети и других внешних эффектов.

## Захват и данные

Основная реализация изолирует сетевой стек приложения в отдельном Linux network namespace — сетевом пространстве имён Linux. Дерево процессов отслеживается через control group version 2 (cgroup v2), а `dumpcap` записывает трафик на выделенной паре виртуальных Ethernet-интерфейсов. Повышенные права получает только короткоживущая вспомогательная программа (helper) с ограниченным протоколом; полная модель привилегий и изоляции принадлежит [правилам безопасности](../common/security.md).

Основной аналитический формат — Apache Parquet, машиночитаемая метаинформация хранится в JavaScript Object Notation (JSON), а конфигурация эксперимента — в Tom's Obvious Minimal Language (TOML). Межстадийные артефакты публикуются атомарно и содержат схему, происхождение, версии и криптографические хеши.

Генерация сначала создаёт абстрактные пакетные события. Детерминированный адаптер может непосредственно построить из них [пакет набора данных](../contracts/dataset.md), не используя Packet Capture Next Generation (PCAPNG) как обязательный промежуточный формат. При необходимости те же события материализуются в PCAPNG или исполняются как активная эмуляция.

Активная эмуляция принимает только уже проверенную последовательность. Она работает в закрытой виртуальной сети без внешнего маршрута и публикует свидетельство изоляции; обязательные условия принадлежат [общей политике безопасности](../common/security.md#сетевая-безопасность-генерации).

## Точки расширения

Средства захвата, декодеры, построители пакетов, извлекатели признаков, модели, генераторы, метрики, валидаторы, оптимизаторы и сериализаторы заменяются через явные контракты. Их семейства перечислены в [каталоге реализаций](../implementations/README.md). Расширение не требует выделять каждый алгоритм в отдельную программу.

Полный путь данных и цикл поиска показаны в [карте конвейера](pipeline.md). Поддерживаемые пределы решения зафиксированы в [границах первой версии](scope.md).
