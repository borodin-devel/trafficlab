# Границы первой версии

## Входит в систему

Первая версия поддерживает:

- запуск Linux-приложения в текущем дистрибутиве Windows Subsystem for Linux version 2 (WSL 2);
- обычные терминальные приложения и Text User Interface (TUI), текстовый пользовательский интерфейс, с сохранением стандартных потоков и управляющего терминала;
- захват входящего и исходящего Internet Protocol (IP) трафика процесса и его потомков;
- завершение по выходу дерева процессов, по времени или по первому из этих событий;
- хранение исходного захвата в Packet Capture Next Generation (PCAPNG);
- потоковое построение нормализованного пакета набора данных;
- обучение нескольких взаимозаменяемых моделей;
- генерацию пакетных событий, прямое построение из них набора данных, необязательную материализацию PCAPNG и активную эмуляцию;
- отдельную проверку исходного и синтетического трафика;
- сравнение несколькими независимыми семействами метрик;
- итеративный подбор параметров генетическим алгоритмом;
- полный, частичный и возобновлённый запуск;
- ограничение параллелизма по памяти и процессору.

## Не входит в систему

Первая версия не поддерживает:

- захват Windows-приложений или процессов другого WSL-дистрибутива;
- использование сетевого пространства имён как полной песочницы для недоверенного приложения;
- гарантированную защиту от привилегированного приложения, способного менять сетевое пространство имён, контрольную группу или сеть;
- точное воспроизведение физического Ethernet- или Wi-Fi-канала;
- анализ полезной нагрузки прикладных протоколов;
- полноценную реализацию стека Transmission Control Protocol (TCP) генератором;
- распределённое обучение на нескольких компьютерах;
- сервисные платформы оркестрации и хранения, включая Kubernetes, Ray, Dask, Celery, Kafka и сервер MLflow.

## Ограничения наблюдения

Виртуальный Ethernet-интерфейс (`veth`) показывает кадры внутри Linux-среды WSL, а не физические кадры адаптера Windows. Поэтому второй канальный уровень — Layer 2 (L2) — означает длину кадра в точке захвата, Ethernet type, адреса виртуальной топологии, направление и временную последовательность.

Механизмы сетевой разгрузки могут менять наблюдаемую форму пакетов. Реализация захвата проверяет и, где возможно, отключает TCP Segmentation Offload (TSO), Generic Segmentation Offload (GSO), Generic Receive Offload (GRO) и Large Receive Offload (LRO). Точные платформенные условия определены в [общих ограничениях](../common/platform.md).

## Ограничения безопасности и данных

Сетевое пространство имён не изолирует файловую систему, процессы, пользователей, межпроцессное взаимодействие и системные вызовы. Повышенные права ограничены отдельной вспомогательной программой (helper), а целевое приложение запускается с исходными правами пользователя.

Полезная нагрузка пакетов по умолчанию не переносится в набор данных. Это уменьшает объём чувствительных данных, но не заменяет контроль доступа. Активная генерация разрешена только в изолированной топологии и при наличии проверяемого свидетельства границы. Нормативные правила находятся в [документе безопасности](../common/security.md).
