# Правила проверки исходного набора

## Назначение

Документ является каноническим перечнем правил проверки пакета по [контракту набора данных](../../contracts/dataset.md). Реализация валидатора ссылается на эти идентификаторы и не определяет параллельный список.

## Схема и значения

| Идентификатор | Проверка |
| --- | --- |
| DATASET-SCHEMA-001 | Версия схемы объявлена и поддерживается выбранным профилем. |
| DATASET-SCHEMA-002 | Все обязательные файлы, таблицы и столбцы присутствуют. |
| DATASET-SCHEMA-003 | Типы, допустимость пустых значений и семантическая версия совпадают с контрактом. |
| DATASET-ENUM-001 | Все перечислимые значения принадлежат множествам, заданным контрактом. |
| DATASET-PRIVACY-001 | Реальные байты полезной нагрузки отсутствуют. |

Неподдерживаемая версия является ошибкой, а не поводом интерпретировать данные по ближайшей известной схеме.

## Индексы и время

| Идентификатор | Проверка |
| --- | --- |
| DATASET-INDEX-001 | `packet_index` уникален во всём следе. |
| DATASET-INDEX-002 | `packet_index` сохраняет порядок пакетов без обратных переходов. |
| DATASET-TIME-001 | `relative_time_ns` неотрицателен. |
| DATASET-TIME-002 | `inter_arrival_ns` неотрицателен, кроме записи с явно отмеченной аномалией временной метки. |

Явно отмеченная аномалия не отменяет проверку типа и единицы времени. Все временные значения остаются целыми наносекундами.

## Длины и направления

| Идентификатор | Проверка |
| --- | --- |
| DATASET-LENGTH-001 | Длины пакета неотрицательны и находятся в диапазонах своих типов. |
| DATASET-LENGTH-002 | Захваченная длина не превышает исходную, а зависимые длины согласованы с ними. |
| DATASET-DIRECTION-001 | Направление каждого пакета имеет известное значение. |
| DATASET-DIRECTION-002 | Доля пакетов с определённым направлением достигает порога профиля. |

Порог покрытия направления является параметром профиля и записывается в `expected` результата правила.

## Ссылочная целостность

| Идентификатор | Проверка |
| --- | --- |
| DATASET-REFERENCE-001 | Каждая ссылка пакета на поток разрешается в таблице потоков. |
| DATASET-REFERENCE-002 | Каждая ссылка пакета и потока на сессию разрешается в таблице сессий. |
| DATASET-REFERENCE-003 | Каждый поток принадлежит ровно одной сессии, а агрегированные границы и счётчики не противоречат связанным строкам. |

## Статистика и целостность файлов

| Идентификатор | Проверка |
| --- | --- |
| DATASET-STATISTICS-001 | Число строк и статистика манифеста совпадают с прочитанными таблицами. |
| DATASET-HASH-001 | Secure Hash Algorithm 256-bit (SHA-256) каждого заявленного файла совпадает с сохранённым значением. |
| DATASET-HASH-002 | Идентификатор пакета соответствует хешируемому составу артефакта. |

## Разделение сессий

| Идентификатор | Проверка |
| --- | --- |
| DATASET-SPLIT-001 | Каждая сессия целиком принадлежит ровно одной из обучающей, проверочной или тестовой частей. |
| DATASET-SPLIT-002 | Пакеты и потоки наследуют часть своей сессии; одна сессия не появляется в нескольких частях. |
| DATASET-SPLIT-003 | Группы, объявленные неделимыми профилем разделения, не пересекают границы частей. |

Нарушение любого правила разделения считается утечкой данных между частями набора и имеет серьёзность `error`.

## Формат результата

Каждое правило создаёт одну запись по [контракту отчёта валидации](../../contracts/validation-report.md). Поля `observed` и `expected` содержат агрегат или короткое свидетельство, но не копируют строки пакетов и не раскрывают исходные адреса.
