# Правила проверки исходного набора

## Назначение

Документ является каноническим перечнем правил проверки пакета по [контракту набора данных](../../contracts/dataset.md). Реализация валидатора ссылается на эти идентификаторы и не определяет параллельный список.

## Матрица правил

Значения базовой серьёзности замкнуты множеством `warning` и `error`, определённым [контрактом отчёта валидации](../../contracts/validation-report.md). Значение «да» в колонке «Обязательно» означает, что профиль не может удалить правило. Колонка «Настраивается» исчерпывающе перечисляет разрешённые параметры; значение «нет» запрещает менять условие или серьёзность правила.

| Идентификатор | Проверка | Обязательно | Базовая severity | Настраивается |
| --- | --- | --- | --- | --- |
| DATASET-SCHEMA-001 | Версия схемы объявлена и поддерживается выбранным профилем. | да | `error` | список поддерживаемых версий |
| DATASET-SCHEMA-002 | Все обязательные файлы, таблицы и столбцы присутствуют. | да | `error` | нет |
| DATASET-SCHEMA-003 | Типы, допустимость пустых значений и семантическая версия совпадают с контрактом. | да | `error` | нет |
| DATASET-ENUM-001 | Все перечислимые значения принадлежат множествам, заданным контрактом. | да | `error` | нет |
| DATASET-PRIVACY-001 | Реальные байты полезной нагрузки отсутствуют. | да | `error` | нет |
| DATASET-INDEX-001 | `packet_index` уникален во всём следе. | да | `error` | нет |
| DATASET-INDEX-002 | `packet_index` сохраняет порядок пакетов без обратных переходов. | да | `error` | нет |
| DATASET-TIME-001 | `relative_time_ns` неотрицателен. | да | `error` | нет |
| DATASET-TIME-002 | `inter_arrival_ns` неотрицателен, кроме записи с явно отмеченной аномалией временной метки. | да | `error` | нет |
| DATASET-LENGTH-001 | Длины пакета неотрицательны и находятся в диапазонах своих типов. | да | `error` | нет |
| DATASET-LENGTH-002 | Захваченная длина не превышает исходную, а зависимые длины согласованы с ними. | да | `error` | нет |
| DATASET-DIRECTION-001 | Направление каждого пакета имеет известное значение. | да | `error` | нет |
| DATASET-DIRECTION-002 | Доля пакетов с определённым направлением достигает порога профиля. | да | `warning` | порог; severity: `warning` или `error` |
| DATASET-REFERENCE-001 | Каждая ссылка пакета на поток разрешается в таблице потоков. | да | `error` | нет |
| DATASET-REFERENCE-002 | Каждая ссылка пакета и потока на сессию разрешается в таблице сессий. | да | `error` | нет |
| DATASET-REFERENCE-003 | Каждый поток принадлежит ровно одной сессии, а агрегированные границы и счётчики не противоречат связанным строкам. | да | `error` | нет |
| DATASET-STATISTICS-001 | Число строк и статистика манифеста совпадают с прочитанными таблицами. | да | `error` | нет |
| DATASET-HASH-001 | Secure Hash Algorithm 256-bit (SHA-256) каждого файла из `manifest.files` совпадает с сохранённым значением. | да | `error` | нет |
| DATASET-HASH-002 | `artifact_id` совпадает с SHA-256 канонического представления, определённого контрактом набора данных. | да | `error` | нет |
| DATASET-SPLIT-001 | Каждая сессия целиком принадлежит ровно одной из обучающей, проверочной или тестовой частей. | да | `error` | нет |
| DATASET-SPLIT-002 | Пакеты и потоки наследуют часть своей сессии; одна сессия не появляется в нескольких частях. | да | `error` | нет |
| DATASET-SPLIT-003 | Группы, объявленные неделимыми профилем разделения, не пересекают границы частей. | да | `error` | определение неделимых групп |

Неподдерживаемая версия является ошибкой, а не поводом интерпретировать данные по ближайшей известной схеме. Явно отмеченная аномалия временной метки не отменяет проверку типа и единицы времени: все временные значения остаются целыми наносекундами. Порог покрытия направления записывается в `expected` результата `DATASET-DIRECTION-002`. Нарушение любого правила разделения считается утечкой данных между частями набора и сохраняет серьёзность `error`.

## Формат результата

Каждое правило создаёт одну запись по [контракту отчёта валидации](../../contracts/validation-report.md). Поля `observed` и `expected` содержат агрегат или короткое свидетельство, но не копируют строки пакетов и не раскрывают исходные адреса.
