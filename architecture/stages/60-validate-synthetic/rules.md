# Правила проверки синтетического трафика

## Назначение

Документ является каноническим перечнем правил проверки packet events — абстрактных пакетных событий — и материализованного синтетического трафика. Реализация валидатора ссылается на эти идентификаторы и не определяет параллельный список.

## Матрица правил

Transmission Control Protocol (TCP) — протокол управления передачей. Internet Protocol (IP) — протокол сетевого уровня. Layer 2 (L2) — канальный уровень. Checksum — контрольная сумма. Endpoint означает конечный сетевой узел. Flow и session означают поток и сессию по [контракту набора данных](../../contracts/dataset.md).

Mode collapse — вырождение генератора, при котором разные входы или случайные выборки дают недостаточно разнообразные результаты. Burst rate — интенсивность всплеска, то есть число пакетов или байтов за заданное скользящее окно.

Базовые значения `severity` ограничены множеством `warning` и `error` из [контракта отчёта валидации](../../contracts/validation-report.md). Значение «да» в колонке «Обязательно» означает, что профиль не может удалить правило. Колонка «Настраивается» исчерпывающе перечисляет параметры, которые разрешено менять.

| --- | --- | --- | --- | --- |
| SYNTH-TIME-001 | Временные метки материализованных пакетов не убывают в порядке `packet_index`. | да | `error` | нет |
| SYNTH-TIME-002 | Интервалы между событиями и материализованными пакетами неотрицательны. | да | `error` | допуск округления часов активной эмуляции |
| SYNTH-LENGTH-001 | Нулевая длина пакета допустима только для явно разрешённой профилем протокольной причины. | да | `error` | перечень причин |
| SYNTH-LENGTH-002 | Длина L2, полная длина IP, длина транспортного заголовка и полезной нагрузки взаимно согласованы. | да | `error` | нет |
| SYNTH-TCP-STATE-001 | Флаги каждого TCP-пакета допустимы в текущем состоянии его потока. | да | `error` | нет |

| SYNTH-ADDRESS-001 | Все разрешённые псевдонимы endpoint отображаются только в адреса настроенного synthetic endpoint space — синтетического адресного пространства. | да | `error` | адресное пространство профиля |

| SYNTH-COUNT-002 | Фактическое число сессий равно числу в запросе генерации. | да | `error` | нет |
| SYNTH-BURST-001 | Интенсивность пакетов и байтов в каждом скользящем окне не превышает физические пределы профиля. | да | `error` | длительность окна; пределы пакетов и байтов |

## Применимость и измерение

Правила TCP получают `skip` только при доказанном отсутствии TCP-потоков. Проверка контрольных сумм учитывает версию IP и правила UDP, но не принимает аппаратную разгрузку как объяснение: синтетический файл проверяется после материализации. Профиль жизненного цикла TCP явно задаёт, требуется ли полное рукопожатие и какое завершение считается допустимым.

Для `SYNTH-BURST-001` отчёт сохраняет максимумы и границы окна. Для `SYNTH-DIVERSITY-001` профиль задаёт признаки, например длину, интервал, направление и переход состояния. Валидатор сравнивает число уникальных комбинаций, энтропию или долю наиболее частого шаблона с порогами профиля; одна метрика не заменяет остальные заявленные профильные проверки.

## Формат результата

Каждое правило создаёт одну запись по [контракту отчёта валидации](../../contracts/validation-report.md). Поля `observed` и `expected` содержат ограниченный агрегат, диапазон или короткое свидетельство. Они не копируют весь поток пакетов и не включают полезную нагрузку.
