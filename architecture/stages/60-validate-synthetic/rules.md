# Правила проверки синтетического трафика

## Назначение

Документ является каноническим перечнем правил проверки packet events — абстрактных пакетных событий — и материализованного синтетического трафика. Реализация валидатора ссылается на эти идентификаторы и не определяет параллельный список.

## Матрица правил

Transmission Control Protocol (TCP) — протокол управления передачей. Internet Protocol (IP) — протокол сетевого уровня. Layer 2 (L2) — канальный уровень. Checksum — контрольная сумма. Endpoint означает конечный сетевой узел. Flow и session означают поток и сессию по [контракту набора данных](../../contracts/dataset.md).

Mode collapse — вырождение генератора, при котором разные входы или случайные выборки дают недостаточно разнообразные результаты. Burst rate — интенсивность всплеска, то есть число пакетов или байтов за заданное скользящее окно.

Базовые значения `severity` ограничены множеством `warning` и `error` из [контракта отчёта валидации](../../contracts/validation-report.md). Значение «да» в колонке «Обязательно» означает, что профиль не может удалить правило, когда указанный в колонке «Применимость» вход применим. Колонка «Настраивается» исчерпывающе перечисляет параметры, которые разрешено менять.

## Применимость и измерение

В `synthetic-events-v1` байтовые правила `SYNTH-LENGTH-*`, `SYNTH-TCP-STATE-*` и `SYNTH-CHECKSUM-001` получают `skip` с причиной `component_not_present`, если `rendered_pcapng` отсутствует; если необязательный компонент присутствует, эти правила применимы. В `synthetic-rendered-v1` и `synthetic-active-v1` отсутствие требуемого пакетного компонента даёт `fail`. Правила TCP получают `skip` с причиной отсутствия TCP-потоков только после успешного чтения применимого пакетного компонента.

Проверка контрольных сумм учитывает версию IP и правила UDP, но не принимает аппаратную разгрузку как объяснение. Профиль жизненного цикла TCP явно задаёт, требуется ли полное рукопожатие и какое завершение считается допустимым.

`SYNTH-NETWORK-001` получает `fail`, если active mode не содержит `isolation-evidence.json`, его хеш или сведения не согласуются с манифестом и наблюдаемой топологией, проверка изоляции неуспешна либо счётчик принятого external egress ненулевой. Это обязательный вход active mode, поэтому `skip` недопустим.

Профиль `synthetic-dataset-v1` для `dataset-validation.json` повторно использует [канонические правила набора данных](../30-validate-source/rules.md), не копируя их идентификаторы или схему. Дополнительное обязательное правило `SYNTH-DATASET-SOURCE-001` требует, чтобы `source.type` был равен `packet_events`, а `source.artifact_id` и `source.sha256` совпадали с идентификатором и хешем `components.packet_events` принятого составного артефакта. Нарушение этого правила имеет `severity = error`.

Для `SYNTH-BURST-001` отчёт сохраняет максимумы и границы окна. Для `SYNTH-DIVERSITY-001` профиль задаёт признаки, например длину, интервал, направление и переход состояния. Валидатор сравнивает число уникальных комбинаций, энтропию или долю наиболее частого шаблона с порогами профиля; одна метрика не заменяет остальные заявленные профильные проверки.

## Формат результата

Каждое правило создаёт одну запись по [контракту отчёта валидации](../../contracts/validation-report.md). Поля `observed` и `expected` содержат ограниченный агрегат, диапазон или короткое свидетельство. Они не копируют весь поток пакетов и не включают полезную нагрузку.
