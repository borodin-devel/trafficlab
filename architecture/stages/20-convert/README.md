# Конвертация захвата

## Назначение

Стадия преобразует закрытый файл Packet Capture Next Generation (PCAPNG) в версионированный пакет набора данных. Основной результат пригоден для потоковой аналитики и не сохраняет полезную нагрузку пакетов по умолчанию.

## Место в конвейере

Стадия выполняется после [захвата](../10-capture/README.md) либо после регистрации внешнего файла PCAPNG. Готовый пакет передаётся [проверке исходного набора](../30-validate-source/README.md).

## Входы

- для результата стадии 10 — только канонический `stages/10_capture/raw/target.pcapng` и успешный отчёт профиля `source-pcap-v1`, чей `validated_artifact_id` совпадает с SHA-256 этого файла;
- для отдельно зарегистрированного внешнего входа — один закрытый и успешно проверенный файл PCAPNG;
- нормализованная конфигурация преобразования;
- выбранный декодер пакетов и его версия;
- сведения о происхождении и хеш входного файла.

## Выходы

Основной режим создаёт и публикует полный каталог `stages/20_convert/dataset/` по [контракту набора данных](../../contracts/dataset.md). До публикации стадия выполняет базовую структурную проверку и записывает её результат во внутренний файл пакета `dataset/validation.json`. Состав файлов, табличные поля и их типы определяет только контракт набора данных.

Внутренняя проверка подтверждает, что стадия может опубликовать структурно целостный пакет. Она не заменяет правила профиля исходных данных следующей стадии: та читает уже опубликованный пакет без изменений и создаёт отдельный отчёт уровня стадии `stages/30_validate_source/validation.json`.

Программа конвертации также поддерживает диагностические и обменные режимы. Для них используются построчный формат JSON Lines (JSONL), основанный на JavaScript Object Notation (JSON), и Comma-Separated Values (CSV):

```text
pcapng -> dataset package
pcapng -> JSONL
pcapng -> CSV
pcapng -> Markdown summary
dataset package -> JSONL sample
dataset package -> Markdown summary
event sequence -> pcapng
```

JSONL и CSV являются вспомогательными экспортами. Они не заменяют пакет набора данных как межстадийный контракт. Преобразование последовательности событий в PCAPNG относится к обратному режиму той же программы, а не к построению исходного набора.

## Основной процесс

Декодер последовательно читает вход и передаёт нормализованные записи ограниченными пакетами. Стадия приводит время к целым наносекундам, присваивает явный индекс пакета и определяет направление. Затем она связывает пакеты с потоками и целыми сессиями, записывает таблицы группами строк и вычисляет статистику и хеши.

При переходе из стадии 10 конвертер не принимает `raw/external.pcapng` или `raw/loopback.pcapng` и не объединяет компоненты самостоятельно. Единственным входом является уже проверенный `raw/target.pcapng`.

В памяти одновременно находятся только ограниченный пакет записей и состояние незавершённых потоков и сессий. Размер пакета задаётся конфигурацией и не зависит от общего размера захвата.

## Функциональные требования

| Идентификатор | Требование |
| --- | --- |
| DAT-001 | Любой принятый файл PCAPNG преобразуется в версионированный пакет набора данных. |
| DAT-002 | Полезная нагрузка пакетов по умолчанию не сохраняется. |
| DAT-003 | Время хранится целым числом наносекунд. |
| DAT-004 | Порядок пакетов сохраняется явно через `packet_index`. |
| DAT-005 | Направление кодируется как `ingress`, `egress`, `internal` или `unknown`. |
| DAT-006 | Для каждого пакета сохраняются длина, временной интервал, протоколы и флаги Transmission Control Protocol (TCP). |
| DAT-007 | Пакеты группируются в потоки и сессии. |
| DAT-008 | Схема набора данных имеет отдельную семантическую версию. |
| DAT-009 | Конвертер работает потоково и не загружает весь захват в память. |
| DAT-010 | Один и тот же вход и одна конфигурация дают детерминированный пакет набора данных. |

## Инварианты

- Порядок записей определяется входом и сохраняется независимо от размера пакета потоковой обработки.
- Один пакет относится ровно к одному потоку и одной сессии.
- Поток относится ровно к одной сессии.
- Разделение на обучающую, проверочную и тестовую части выполняется по целым сессиям.
- Конфигурация, версия декодера и хеш входа входят в происхождение результата.
- Поля и допустимые значения результата соответствуют [контракту набора данных](../../contracts/dataset.md).

## Ошибки и восстановление

Ошибка декодера, неизвестная обязательная схема, невозможность завершить группу или несоответствие записанных счётчиков завершает стадию без публикации результата. Повторный запуск создаёт новое временное выполнение по [общему жизненному циклу артефакта](../../common/artifact-lifecycle.md).
