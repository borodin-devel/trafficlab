# Обучение моделей

## Назначение

Стадия обучает одну или несколько моделей трафика на проверенном пакете набора данных. Выбор реализации отделён от общего процесса обучения.

## Место в конвейере

Стадия выполняется после [проверки исходного набора](../30-validate-source/README.md). Результат передаётся стадии генерации.

## Входы

- неизменяемый пакет по [контракту набора данных](../../contracts/dataset.md), допущенный проверкой;
- конфигурация с `model.type` и параметрами выбранной реализации;
- seed — исходное значение генератора псевдослучайных чисел;
- resource budget — заданный предел вычислительных ресурсов.

Каждая реализация объявляет требования к feature schema — схеме входных признаков. До обучения стадия проверяет совместимость схемы набора с этими требованиями.

## Выходы

Каждое успешное обучение публикует самостоятельный каталог по [контракту артефакта модели](../../contracts/model-artifact.md). Каталог содержит данные модели, сведения о происхождении обучения, метрики и результат валидации.

## Основной процесс

Стадия читает конфигурацию, находит реализацию по `model.type` и проверяет её требования к признакам. Затем она передаёт реализации только пакет набора данных, конфигурацию, seed и resource budget. Реализация обучает модель и передаёт результат издателю артефакта.

Несколько независимых обучений могут выполняться параллельно, если их совокупные ограничения укладываются в доступный resource budget. Планировщик не запускает работу без выделенных ей ресурсов.

Seed и сведения об окружении сохраняются по [правилам воспроизводимости](../../common/reproducibility.md). Выделение resource budget и параллелизм подчиняются [общей политике ресурсов](../../common/resources.md). Независимость стадии от внутреннего состояния других стадий определена [правилами оркестрации](../../common/orchestration.md).

## Инварианты

В формулировках требований dataset package означает пакет набора данных, capture stage — стадию захвата, а model artifact directory — каталог артефакта модели. Dataset hash означает хеш набора данных, schema version — версию схемы, а Python pickle — встроенный в Python формат сериализации объектов.

| ID | Требование |
| --- | --- |
| MOD-001 | Модель выбирается через `model.type` в конфигурации. |
| MOD-002 | Каждая модель объявляет требования к feature schema. |
| MOD-003 | Обучение получает только dataset package, конфигурацию, seed и resource budget. |
| MOD-004 | Обучение не обращается напрямую к capture stage. |
| MOD-005 | Результат обучения — самостоятельный model artifact directory. |
| MOD-006 | Model artifact содержит dataset hash, schema version, параметры и seed. |
| MOD-007 | Формат model artifact не должен требовать небезопасной десериализации Python pickle. |
| MOD-008 | Несколько моделей могут обучаться параллельно при наличии ресурсов. |

## Ошибки и восстановление

Несовместимая feature schema, неизвестный `model.type`, недостаточный resource budget или ошибка обучения завершают конкретное обучение как неуспешное. Частичный результат не публикуется как готовый артефакт. Повторный запуск использует тот же неизменяемый вход и явно заданный seed либо новую конфигурацию.
