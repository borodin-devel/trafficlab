# Управление процессами захвата

## Назначение

Документ задаёт общие правила control group version 2 (cgroup v2), launcher, сигналов, прав пользователя и Text User Interface (TUI), текстового пользовательского интерфейса.

## Дерево процессов

Перед запуском создаётся отдельная cgroup. Launcher вступает в неё до `execve`, поэтому приложение и потомки наследуют принадлежность без промежутка между стартом и учётом. Поле `populated` в `cgroup.events` определяет, остались ли живые процессы во всём поддереве.

Код завершения главного процесса сохраняется отдельно. Он не доказывает, что потомки завершились, и не определяет статус стадии.

## Launcher и права

Команда передаётся массивом аргументов без shell. Launcher вступает в подготовленное сетевое окружение, настраивает группу процессов и терминал, переходит в рабочий каталог, устанавливает разрешённое окружение, затем сбрасывает дополнительные группы, Group Identifier (GID), User Identifier (UID) и capabilities. Целевое приложение никогда не работает от root.

Граница привилегий и проверка аргументов определены в [общих правилах безопасности](../../common/security.md).

## Сигналы и завершение

Controller отслеживает cgroup, а не только Process Identifier (PID) главного процесса. При ограничении времени всему дереву сначала отправляется `SIGTERM`. После льготного периода оставшиеся процессы завершаются через `cgroup.kill` или `SIGKILL`. Выбор момента и режимы принадлежат [жизненному циклу](lifecycle.md#режимы-завершения).

Прерывание с терминала проходит через тот же управляемый путь. Сигналы средства записи отделены от сигналов приложения.

## TUI

Интерактивному приложению передаются исходные файловые дескрипторы терминала. Стандартные потоки не проксируются через Python pipes. Группа процессов приложения временно становится foreground-группой терминала.

Controller не читает стандартный ввод и не пишет сообщения поверх интерфейса. Его журналы уходят в файл. После завершения он восстанавливает настройки терминала; средство записи работает в отдельной группе процессов.

## Инварианты

- Launcher вступает в cgroup до запуска целевой команды.
- Потомки учитываются независимо от выхода главного процесса.
- Основная программа и целевое приложение не получают повышенные права.
- Один путь сигналов управляет штатным завершением, timeout и прерыванием.
- Управляющий терминал возвращается controller после любого исхода.
