# Захват трафика

## Назначение

Стадия запускает заданное приложение в контролируемой сетевой среде и записывает его пакетный трафик в Packet Capture Next Generation (PCAPNG). Результат сохраняет связь с командой, процессом и фактической конфигурацией среды.

## Место в конвейере

Стадия выполняется после успешной [предварительной проверки](../00-preflight/README.md). Опубликованный захват становится входом стадии конвертации.

## Входы

- команда приложения как массив аргументов, рабочий каталог и разрешённые переменные окружения;
- режим и пределы выполнения;
- параметры сетевой среды и пакетного захвата;
- успешный отчёт предварительной проверки;
- выбранная реализация backend захвата.

## Выходы

Стадия создаёт каталог `stages/10_capture/` по [контракту каталога запуска](../../contracts/run-directory.md). Он содержит состояние стадии, описание команды и процесса, фактическую сетевую и capture-конфигурацию, статистику, отчёт проверки, журналы и один или несколько файлов `.pcapng`. Состав и условия готовности определяет [публикация](publication.md).

## Основной процесс

Порядок подготовки, запуска, завершения и очистки определён в [жизненном цикле](lifecycle.md). Отдельные владельцы описывают [сетевую среду](network-environment.md), [управление процессами](process-control.md), [запись пакетов](packet-capture.md) и [публикацию результата](publication.md).

## Функциональные требования

| ID | Требование |
| --- | --- |
| CAP-001 | Команда приложения принимается как массив аргументов без интерпретации shell. |
| CAP-002 | Приложение запускается в отдельном Linux network namespace — сетевом пространстве имён Linux. |
| CAP-003 | Потомки приложения наследуют network namespace. |
| CAP-004 | Приложение и потомки помещаются в отдельную control group version 2 (cgroup v2). |
| CAP-005 | Захватывается только трафик, проходящий через выделенный виртуальный Ethernet-интерфейс (`veth`) данного запуска. |
| CAP-006 | Поддерживаются Internet Protocol version 4 (IPv4) и Internet Protocol version 6 (IPv6), если они доступны в выбранной конфигурации Windows Subsystem for Linux (WSL). |
| CAP-007 | Сохраняются входящие и исходящие пакеты. |
| CAP-008 | Результат записывается в `.pcapng`. |
| CAP-009 | Захват начинается до запуска приложения. |
| CAP-010 | Захват прекращается после выхода всех процессов cgroup. |
| CAP-011 | В режиме ограничения времени дерево процессов завершается сначала штатным сигналом, затем принудительно. |
| CAP-012 | Код завершения главного процесса сохраняется независимо от результата стадии захвата. |
| CAP-013 | Для Text User Interface (TUI), текстового пользовательского интерфейса, управляющий терминал передаётся приложению без проксирования через Python pipes. |
| CAP-014 | Все настройки namespace, маршрутизации, Domain Name System (DNS) и offload сохраняются в манифесте. |
| CAP-015 | Незавершённый `.pcapng` не публикуется как валидный артефакт. |

## Инварианты

- До запуска приложения запись пакетов готова, процессная группа создана, а сетевая среда подготовлена.
- Во время выполнения выделенной сетевой средой пользуется только исследуемое дерево процессов.
- После выполнения запись штатно закрыта, выходы проверены, а созданные системные ресурсы удалены.
- Код завершения приложения и статус стадии являются разными результатами.
- Только `completed` в `stage.json` разрешает использовать выходы дальше.

## Ошибки и восстановление

Ошибка приложения не отменяет закрытие захвата и очистку. Ошибка записи, проверки или очистки запрещает публикацию успешного результата. Повторный запуск создаёт новое временное выполнение по [общему жизненному циклу артефакта](../../common/artifact-lifecycle.md).
