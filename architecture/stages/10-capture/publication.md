# Публикация захвата

## Назначение

Документ определяет состав результата стадии, первичную проверку, состояние и условия публикации. Алгоритм проверки Packet Capture Next Generation (PCAPNG) принадлежит [валидатору](../../implementations/validators/packet-capture.md).

## Состав результата

Каталог `stages/10_capture/` содержит:

```text
stage.json
target.json
network.json
statistics.json
validation.json
raw/*.pcapng
logs/
```

`target.json` фиксирует массив аргументов, рабочий каталог, User Identifier (UID), Group Identifier (GID), код завершения главного процесса и причину остановки. `network.json` хранит фактические адреса, маршруты, Domain Name System (DNS), режим Windows Subsystem for Linux (WSL) и offload. `statistics.json` хранит итоговые счётчики записи. Журналы включают отдельный stderr средства захвата.

Точные обязательные корневые пути задаёт [контракт каталога запуска](../../contracts/run-directory.md). Незаявленные временные файлы не становятся публичными выходами.

## Первичная проверка

После штатного закрытия файлов стадия вызывает валидатор на каждом объявленном `.pcapng` и на итоговом объединённом файле, если он создаётся. Результат содержит структурную корректность, пригодность набора данных, причины отказа и вычисленные хеши.

Непригодный для обучения, но структурно корректный пустой захват может быть сохранён для диагностики. Он не публикуется как пригодный вход конвертации. Политика минимального числа пакетов задаётся конфигурацией запуска.

## Условия публикации

Статус `completed` допустим только когда:

- средство записи успешно завершилось и закрыло все заявленные файлы;
- структурная проверка каждого обязательного файла успешна;
- статистика и манифесты согласованы с файлами;
- вычислены хеши всех выходов;
- обязательная очистка системных ресурсов завершилась;
- полный каталог опубликован по [общему жизненному циклу артефакта](../../common/artifact-lifecycle.md).

Код завершения приложения не обязан быть нулём. Он сохраняется независимо и сам по себе не делает захват повреждённым.

## Отказ и диагностика

Незакрытый, нечитаемый или несогласованный файл оставляет стадию `failed` либо `invalid`. Временный каталог может сохраняться для диагностики, но не становится входом следующей стадии. Уже опубликованный артефакт не изменяется при повторной попытке.

## Инварианты

- Наличие `.pcapng` не означает готовность результата.
- Проверка и хеширование предшествуют публикации.
- `stage.json` является источником истины о готовности.
- Публикуется полный набор выходов одной атомарной операцией.
