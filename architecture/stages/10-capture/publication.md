# Публикация захвата

## Назначение

Документ определяет состав результата стадии, первичную проверку, состояние и условия публикации. Алгоритм проверки Packet Capture Next Generation (PCAPNG) принадлежит [валидатору](../../implementations/validators/packet-capture.md).

## Состав результата

Каталог `stages/10_capture/` содержит:

```text
stage.json
target.json
network.json
statistics.json
validation.json
raw/
  target.pcapng
  external.pcapng  # необязательно
  loopback.pcapng  # необязательно
logs/
```

`target.json` фиксирует массив аргументов, рабочий каталог, User Identifier (UID), Group Identifier (GID), код завершения главного процесса и причину остановки. `network.json` хранит фактические адреса, маршруты, Domain Name System (DNS), режим Windows Subsystem for Linux (WSL) и offload. `statistics.json` хранит итоговые счётчики записи. Журналы включают отдельный stderr средства захвата.

`raw/target.pcapng` обязателен для каждого успешного захвата и является единственным каноническим пакетным артефактом стадии. `raw/external.pcapng` и `raw/loopback.pcapng` могут сохраняться как свидетельства происхождения, но не являются самостоятельными межстадийными входами. Сборка канонического файла принадлежит выбранной реализации захвата; основная реализация описана в [документе `netns-dumpcap`](../../implementations/capture-backends/netns-dumpcap.md#сборка-канонического-артефакта).

Точные обязательные корневые пути задаёт [контракт каталога запуска](../../contracts/run-directory.md). Незаявленные временные файлы не становятся публичными выходами.

## Первичная проверка

После штатного закрытия компонентов и успешной сборки `raw/target.pcapng` стадия ровно один раз вызывает валидатор для этого канонического файла. Валидатор записывает `validation.json` по [контракту отчёта валидации](../../contracts/validation-report.md) с профилем `source-pcap-v1`, результатами стабильных правил, причинами отказа, вычисленными хешами и производной `summary`. `validated_artifact_id` равен SHA-256 канонического `raw/target.pcapng`. Проверки необязательных компонентов могут входить в свидетельства правил, но не меняют идентичность проверяемого артефакта. Владельцем вычисления `summary.structurally_valid` и `summary.dataset_usable` является [валидатор первичной проверки](../../implementations/validators/packet-capture.md), а этот документ владеет правилом допуска в следующую стадию.

Структурно корректный, но непригодный пустой захват получает `summary.dataset_usable = false` и не может завершиться со `status = completed`. Его временный каталог может сохраняться для диагностики. Политика минимального числа пакетов задаётся конфигурацией запуска.

## Условия публикации

Статус `completed` допустим только когда:

- средство записи успешно завершилось и закрыло все заявленные файлы;
- канонический `raw/target.pcapng` успешно собран и существует;
- единственный отчёт `validation.json` имеет `status = pass` или `pass_with_warnings`, профиль `source-pcap-v1`, `summary.dataset_usable = true` и `validated_artifact_id`, совпадающий с SHA-256 `raw/target.pcapng`;
- статистика и манифесты согласованы с файлами;
- вычислены хеши всех выходов;
- обязательная очистка системных ресурсов завершилась;
- полный каталог опубликован по [общему жизненному циклу артефакта](../../common/artifact-lifecycle.md).

Код завершения приложения не обязан быть нулём. Он сохраняется независимо и сам по себе не делает захват повреждённым.

Стадия 20 получает только `raw/target.pcapng`; компоненты `raw/external.pcapng` и `raw/loopback.pcapng` в gate не входят. Ни наличие компонентов, ни отдельные свидетельства об их проверке не заменяют канонический файл и связанный с ним успешный отчёт.

## Отказ и диагностика

Невозможность закрыть заявленный компонент, собрать или проверить канонический `raw/target.pcapng`, несовпадение его хеша с `validated_artifact_id` либо `summary.dataset_usable = false` оставляет стадию `failed` или `invalid` и запрещает публикацию со `status = completed`. Временный каталог может сохраняться для диагностики, но не становится входом следующей стадии. Уже опубликованный артефакт не изменяется при повторной попытке.

## Инварианты

- Наличие компонента `.pcapng` не означает готовность результата; межстадийным артефактом является только `raw/target.pcapng`.
- Проверка и хеширование предшествуют публикации.
- `stage.json` является источником истины о завершении стадии, а канонические `rules[]` в `validation.json` — о структурной корректности и пригодности данных; `summary` только производна от них.
- Публикуется полный набор выходов одной атомарной операцией.
