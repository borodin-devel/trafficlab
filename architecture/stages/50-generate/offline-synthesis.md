# Автономный синтез

## Назначение

Offline synthesis — автономная обработка проверенных packet events — абстрактных пакетных событий — без отправки пакетов в сеть. Процесс всегда строит нормализованный dataset package — пакет набора данных — и при необходимости отдельно материализует файл Packet Capture Next Generation (PCAPNG).

## Адаптер событий в набор данных

Канонический владелец преобразования проверенной последовательности событий в синтетический пакет набора данных — эта стадия. Детерминированный адаптер принимает неизменяемый артефакт `packet_events`, проверяет его идентификатор и хеш и создаёт полный пакет по [контракту набора данных](../../contracts/dataset.md). Он напрямую отображает семантику событий в нормализованные таблицы, детерминированно выводит потоки, сессии и статистику и сохраняет происхождение от артефакта событий.

В манифесте набора `source.type` равен `packet_events`, а `source.artifact_id` и `source.sha256` указывают на входной артефакт событий. Этот путь не создаёт PCAPNG, не строит байты пакетов и не вызывает packet decoder — декодер пакетов. Полученный идентификатор и хеш набора записываются в компонент `dataset` [составного синтетического манифеста](../../contracts/packet-events.md#составной-синтетический-артефакт). Именно этот компонент получает [стадия 70](../70-compare/README.md); она не читает события или PCAPNG напрямую.

При одинаковых событиях, версии схемы и конфигурации адаптера получается один и тот же пакет. Несогласованность ссылок между событиями, потоками или сессиями, невозможность вывести обязательное поле либо несовпадение количества строк оставляет временный пакет неопубликованным и сохраняет диагностику.

## Профиль проверки

Опубликованный offline-результат всегда содержит `packet_events` и `dataset`. Без материализованных пакетов он передаётся стадии 60 с профилем `synthetic-events-v1`; после добавления `rendered_pcapng` — с профилем `synthetic-rendered-v1`. Активная эмуляция выбирает `synthetic-active-v1` и добавляет обязательные `active_capture` и `isolation_evidence`. Состав профилей и правила их применимости принадлежат [стадии 60](../60-validate-synthetic/README.md#профили); эта стадия их только выбирает по составу выхода.

## Материализация PCAPNG

## Процесс

```text
проверенная последовательность событий
  -> разрешение синтетических endpoint
  -> построение протокольных пакетов
  -> потоковая запись
  -> закрытие synthetic.pcapng
```

Стадия разрешает псевдонимы endpoint — конечных сетевых узлов — по конфигурации синтетического адресного пространства. Затем выбранное [средство построения пакетов](../../implementations/packet-renderers/scapy.md) создаёт кадры Ethernet и пакеты Internet Protocol (IP), Transmission Control Protocol (TCP) или User Datagram Protocol (UDP). Средство вычисляет зависимые длины и контрольные суммы.

Пакеты записываются по одному или ограниченными группами. Полная последовательность и готовый захват не загружаются в память целиком. Файл закрывается до вычисления хеша и передачи [проверке синтетического трафика](../60-validate-synthetic/README.md).

## Область применения

Нормализованный набор используется для массовой оценки кандидатов, сравнения, обучения последующих моделей и эволюционного поиска. PCAPNG материализуется там, где нужна проверка реальных байтов представления; он не является обязательным промежуточным форматом для сравнения наборов.

## Ошибки

Неразрешимый endpoint, недопустимый протокольный переход, ошибка построения пакета или записи оставляет временный файл неопубликованным. Уже опубликованная последовательность событий не изменяется. Повторная материализация использует тот же вход, конфигурацию renderer и версию реализации.
