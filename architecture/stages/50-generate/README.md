# Генерация синтетического трафика

## Назначение

Стадия создаёт синтетическую последовательность трафика. Общим результатом всех режимов является packet event — абстрактное пакетное событие. Последующая обработка либо материализует события в файл Packet Capture Next Generation (PCAPNG), либо воспроизводит их между изолированными endpoint — конечными сетевыми узлами.

## Место в конвейере

Стадия выполняется после [обучения](../40-train/README.md) для режима `model` либо получает проверенный пакет набора данных для независимых режимов. Результат передаётся [проверке синтетического трафика](../60-validate-synthetic/README.md).

## Входы

- запрос генерации с режимом, числом потоков и сессий, пределами длительности и интенсивности;
- фиксированный seed — исходное значение генератора псевдослучайных чисел;
- [артефакт модели](../../contracts/model-artifact.md) для режима `model`;
- проверенный [пакет набора данных](../../contracts/dataset.md) для режима `replay-perturbed` и настраиваемых эмпирических распределений;
- конфигурация профиля протоколов и синтетических endpoint.

Каждый режим принимает только нужное ему подмножество входов. Отсутствие необязательного для режима артефакта не является ошибкой.

## Выходы

Все режимы создают версионированную последовательность по [контракту пакетных событий](../../contracts/packet-events.md). Автономный синтез дополнительно создаёт закрытый `synthetic.pcapng`. Активная эмуляция создаёт захват наблюдавшегося обмена между двумя endpoint.

Результаты и их хеши регистрируются в каталоге `stages/50_generate/` по [контракту каталога запуска](../../contracts/run-directory.md). Частичный файл или незавершённый обмен не публикуется как готовый результат.

## Основной процесс

Стадия выбирает одну из четырёх равноправных реализаций: `model`, `random`, `replay-perturbed` или `active-emulation`. Реализация получает нормализованный запрос и seed и создаёт упорядоченные абстрактные события. Формат событий не зависит от выбранной реализации.

Независимая семантическая проверка подтверждает порядок, ссылки, допустимые протокольные переходы и ограничения запроса. Только прошедшая её последовательность передаётся [автономному синтезу](offline-synthesis.md) или [активной эмуляции](active-emulation.md).

Seed и версии реализаций сохраняются по [правилам воспроизводимости](../../common/reproducibility.md). Ограничения сетевой изоляции определяет [общая политика безопасности](../../common/security.md).

## Инварианты

В формулировках требований renderer означает средство построения пакетов, checksum — контрольную сумму, а baseline — исходный вариант для сравнения без обученной модели. Ethernet — семейство технологий канального уровня, Internet Protocol (IP) — протокол сетевого уровня, Transmission Control Protocol (TCP) и User Datagram Protocol (UDP) — протоколы транспортного уровня.

| ID | Требование |
| --- | --- |
| GEN-001 | Модель генерирует не байты пакетов, а абстрактные packet events. |
| GEN-002 | Packet events проходят независимую семантическую валидацию. |
| GEN-003 | Renderer преобразует события в Ethernet/IP/TCP/UDP-пакеты. |
| GEN-004 | Renderer рассчитывает длины, checksums и зависимые поля. |
| GEN-005 | Результат автономного синтеза записывается в `.pcapng`. |
| GEN-006 | Генератор поддерживает фиксированный seed. |
| GEN-007 | Генератор поддерживает случайный baseline без обученной модели. |
| GEN-008 | Активное воспроизведение соблюдает [границу сетевой изоляции](../../common/security.md#сетевая-безопасность-генерации). |
| GEN-009 | Генерируемые адреса соблюдают [ограничения синтетического адресного пространства](../../common/security.md#сетевая-безопасность-генерации). |

## Ошибки и восстановление

Неизвестный режим, несовместимый вход, нарушение контракта события или семантической проверки завершает конкретную генерацию как неуспешную. Ошибка материализации либо активной эмуляции сохраняет диагностику, но не меняет уже проверенную последовательность событий. Повторный запуск использует те же неизменяемые входы и seed либо новый запрос.
