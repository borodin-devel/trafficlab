# Предварительная проверка

## Назначение

Стадия проверяет, что среда готова к запуску конвейера. Она не изменяет сеть и не запускает целевое приложение.

## Место в конвейере

Предварительная проверка выполняется первой. Успешный результат разрешает начать [захват](../10-capture/README.md).

## Входы

- нормализованная конфигурация запуска;
- выбранная реализация захвата;
- требования реализации к правам и ресурсам;
- фактическая среда Linux.

## Выходы

Каталог `stages/00_preflight/` содержит `stage.json`, `system.json` и `validation.json` по [контракту каталога запуска](../../contracts/run-directory.md). В отчёте фиксируются найденные версии, фактические возможности и причины отказа.

## Основной процесс

Проверяются:

1. запуск внутри Windows Subsystem for Linux version 2 (WSL 2);
2. наличие control group version 2 (cgroup v2), включая `/sys/fs/cgroup/cgroup.controllers`;
3. наличие конкретных внешних команд, требуемых выбранной реализацией; для основной реализации это `ip`, `nft`, `ethtool`, `dumpcap`, `capinfos` и `tshark`, а при включённом loopback-захвате с объединением файлов — также `mergecap`;
4. возможность получить только необходимые повышенные права и выполнить пробные операции реализации;
5. свободное место на диске с учётом лимита захвата;
6. доступная внутри WSL память и возможность зарезервировать заявленные ресурсы;
7. состояние `systemd` и системных служб, от которых зависит реализация;
8. режим сети WSL, доступность Internet Protocol version 4 (IPv4) и Internet Protocol version 6 (IPv6), внешней сети и требуемых входящих соединений;
9. доступность выбранного backend Domain Name System (DNS) и ожидаемый путь DNS-трафика;
10. отсутствие конфликта выбранной подсети с маршрутами WSL, Virtual Private Network (VPN), контейнеров и параллельных запусков;
11. возможность проверить и, если требуется, отключить offload-функции экспериментальных интерфейсов.

Проверка ищет команды, а не названия пакетов дистрибутива. Платформенные ожидания определены в [общих ограничениях](../../common/platform.md), а бюджет — в [правилах ресурсов](../../common/resources.md).

## Инварианты

- Фактический режим сети и доступные семейства адресов записаны в результат.
- Предупреждение не маскирует отсутствие обязательной возможности.
- Проверка прав не расширяет границу привилегий основной программы.
- Неуспешный результат не разрешает запуск зависимой стадии.

## Ошибки и восстановление

Отсутствующая команда, недостаток ресурса, конфликт подсети или невозможность безопасно настроить среду завершают стадию со статусом `failed` либо `invalid`. После исправления среды создаётся новое выполнение стадии; старый отчёт сохраняется для диагностики.
